# Code of Conduct

## Standard

TPMN-PSL is a specification project. All interactions — issues, PRs, discussions — are evaluated on technical merit against the spec text.

**Expected:**
- Reference specific spec sections when proposing changes
- Ground claims with evidence (the spec's own standard applies to its community)
- Disagree constructively — critique the argument, not the person
- Respect that domain expertise varies; ask before assuming

**Not acceptable:**
- Personal attacks, harassment, or discrimination of any kind
- Dismissing contributions based on credentials rather than content
- Publishing others' private information without consent
- Sustained disruption of discussions

## Scope

This applies to all project spaces: GitHub issues, PRs, discussions, and any venue where you represent this project.

## Enforcement

Maintainers may remove, edit, or reject contributions that violate this code. Repeated violations result in temporary or permanent bans.

Reports: open a GitHub issue labeled `conduct` or email the maintainer directly.
